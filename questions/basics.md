0. - "浮点数算不准" 是怎么造成的? 为什么许多语言中, `1.0 - 0.9` 的计算值与理论值有差异, 而 `1.0 - 0.75` 的计算值则与理论值一致呢?
    > jshell
    ```jshell
    jshell> 1.0-0.9
    $1 ==> 0.09999999999999998
   ```
   
   > javascript
   ```javascript
    > 1.0-0.9
    < 0.09999999999999998
    ```
   
   > python3
   ```python3
    >>> 1.0-0.9
    0.09999999999999998   
    ```
   
   > go-shell
   ```go
    $ var a, b = 1.0, 0.9
    $ println(a - b)
   
   +1.000000e-001
    ```
   
    你能在草稿纸上演算计算机计算`1.0 - 0.9` 的过程吗? (不同硬件, 计算机系统, 语言 对存储和计算过程的处理有所差异, 演算过程合理即可)
